# Session — Workflow v2 Hardening Rollout

## Context

Started from a clean main on steaksoap. Owner shared the HDVA audit (132
commits on main, 1000+ line pages, sub-fed memory, orphan agents) and asked
for the same structural fixes upstream in the template. Reviewed three repos
the owner cited (karpathy-skills, claude-mem, anthropics/claude-code) — kept
the Karpathy principles, rejected claude-mem's runtime stack, designed our
own tagged memory inspired by it.

## What was built

All on `feat/workflow-v2`. Two commits:

1. `feat(workflow): install mechanical floor + tagged memory system` (79876fa)
2. `feat(workflow): add hooks, delegation, sizing rule, release journaling` (52b9c1e)

Highlights:
- Anti-main pre-commit gate (`.husky/pre-commit`)
- 5 always-loaded rules: `critical.md`, `memory-protocol.md`, `principles.md`,
  `releases.md` (added) + `workflow.md` (existing, broad path)
- `.claude/memory/` skeleton + `TAGS.md` (closed vocabulary) + `INDEX.md`
  auto-generator + `README.md`
- `decisions.md` migrated to `decisions/2026-04-16-template-baseline.md`

  upstream overwrite (backup/restore on merge)
- `.claude/settings.json` with 3 hooks (SessionStart, UserPromptSubmit, Stop)

- `/delegate` and `/integrate` commands for Sonnet parallel sessions
- `git.md` enriched commit body template (WHY/WHAT/IMPACT/TEST)
- `commit-msg` hook warns (non-blocking) on missing WHY for feat/fix
- `.release-it.json` `after:release` hook auto-writes a memory entry per release
- `health-check.md` Phase 5.5 detects docs/.claude/ drift
- `eslint.config.js` adds `max-lines` / `max-lines-per-function` warnings

## Frictions encountered

1. **Indexer bug**: `e.type` from frontmatter (`decision` singular) overwrote
   the folder name (`decisions`). Fixed by renaming to `e.folder`.
2. **`.gitignore` excluded `.claude/memory/`** — added `!.claude/memory/`
   and `!.claude/settings.json` to the whitelist.
3. **First sizing warning fired**: `Select.tsx` 173-line render function.
   Resolution: relaxed `max-lines-per-function` to 200 for `src/components/`
   (stateful atoms legitimately need it) while keeping 150 for `src/pages/`
   (the actual HDVA target).
4. **`pnpm lint` uses `--max-warnings 0`** — sizing "warn" effectively becomes
   blocking in `validate`. Documented in `sizing.md` and the commit body.
   Acceptable: matches the template's strict policy; can be relaxed later.

## Decisions taken

- Memory directory **tracked in Git**, not gitignored. Owner wants
  "psychopath-level" traceability.
- Hooks are **augmentation, not enforcement**. Primary enforcement = husky +
  always-loaded rules. Hooks add SessionStart context + per-prompt reminders +
  end-of-session nudge.
- Migration strategy: **single dated import file** for legacy `decisions.md`
  rather than re-ADR per entry (low value, high noise).
- ESLint sizing: warn level, scoped paths. Acclimate, then upgrade to error
  in P2.
