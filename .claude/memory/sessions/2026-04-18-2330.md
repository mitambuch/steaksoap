## Release pipeline frictions rencontrées

- **Playwright browsers non installés** (premier release attempt) —
  `pnpm exec playwright install` requis. Dépense one-shot ~170 MiB.
- **WebKit a11y failure pré-existante** — 2 tests sur 84 rouge : `<p>`
  avec text-muted #8a8a8a rendu à ~#757575 par WebKit, contrast 4.29
  sous seuil 4.5 WCAG AA. Fix : bump `--color-muted` à #a0a0a0
  (contrast ~7.4 avec margin de sécurité). Shipped comme `fix(a11y)`
  séparé dans le cycle v5.5.0.
- **ESLint parseError sur studio/** — root ESLint essayait de parser
  les fichiers studio avec le tsconfig root. Fix : `studio` ajouté à
  globalIgnores de `eslint.config.js`.
- **lint-staged warnings** — ESLint sort des warnings "File ignored"
  sur les fichiers studio ignorés, qui deviennent erreurs avec
  `--max-warnings 0`. Fix : ajout de `--no-warn-ignored` au hook.
- **Husky bloque commit sur main** — correctement (rule #1) — la
  commit auto-générée par `memory-record-release.js` passe parce que
  release-it bypass le hook. Mais ma commit manuelle du memory wrap
  doit passer par une branche dédiée.
