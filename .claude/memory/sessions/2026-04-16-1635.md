## Frictions encountered

1. **Windows env-var case-insensitivity bug** in ensure-pnpm tests: pnpm
   injects `NPM_CONFIG_USER_AGENT` (uppercase); my lowercase `delete`
   missed it, the uppercase leaked to the spawned child, test failed with
   exit 0 instead of 1. Fix: strip ALL case-variants before reinstating.
2. **Regex `\s*$` in sync-reference-counts** ate the blank line after the
   heading on first write. Diagnosed via diff inspection. Fix: explicit
   `[ \t]*` instead of `\s*` (JS `\s` matches `\n` regardless of `m` flag).
3. **Initial syntax error** in sync-ref script: unescaped backticks
   inside a template literal. Caught by `node -c` before committing.
4. **Uncommitted release journals** from v5.2.2 and v5.3.0 blocked the
   v5.3.1 release with "working dir dirty". Caught them, committed on a
   catch-up branch, retried release. Pattern: after every release, check
   `git status` before the next release gate.

## Decisions taken

- **`STEAKSOAP_TEST_ROOT` env var** pattern for script testability. Minimal
  refactor (top-of-file 1-liner), works for any script that resolves paths
  relative to PATHS.root. Will reuse for any future script tests.
- **Spawned-child test pattern** over module-import tests. Scripts run
  with side effects at top level (not exporting main()); spawning treats
  them as black boxes, same as how they run in CI. Higher fidelity, minor
  speed cost (~4s per test via vitest).
- **`docs:sync:check` before test:coverage** in `pnpm validate`: fast
  check (no test runner warmup), fails early if docs drifted.
