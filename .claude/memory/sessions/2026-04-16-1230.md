## Validation

`pnpm validate:full` green before the release gate fired. 82 passed, 2 skipped
(webkit Tab), 0 failed across 84 tests on 3 browsers. Unit coverage still
90.15% / 81.26% / 92.37%. Build still ~4.9s.

## Frictions

- **Accidentally bundled two logical changes in one commit** (preinstall +
  validate:full). Caught it during review, used `git reset --soft HEAD~1`
  + temporary edit roll-back to split into two atomic commits. Good
  recovery but the discipline was the right call — future Git log stays
  clean. Lesson: stage each logical change separately from the start.

- **Release took 284s** (validate:full is slow because of E2E on 3 browsers).
  Acceptable for a release gate; would be painful on every commit. That's
  why we kept `validate` = fast, `validate:full` = slow-release-only.

## Decisions taken

- **WebKit Tab tests skip, not fix**: the test assumption is browser-level,
  not app-level. Fix would require driving a WebKit system preference which
  is out of Playwright's API surface. Skip is the right call.

- **`validate` stays fast, `validate:full` is release-only**: splitting was
  the cheapest way to keep daily DX snappy while plugging the release hole.

- **TAGS.md template-ownership made explicit**: clients cannot add tags
  locally (will be wiped on `base:update`). To extend the vocabulary, PR
  the template. This was always the de-facto rule; now it's documented.
