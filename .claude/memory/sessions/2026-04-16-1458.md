## What shipped in v5.2.1

Branch `fix/audit-batch-3` → 5 commits (one docs, four fix):

1. **fix(a11y)** — Accordion uses `inert={!isOpen}` (React 19 native boolean)
   to block Tab focus on closed panels. `aria-hidden` alone doesn't block
   focus; `inert` does both focus-blocking and a11y-hiding.
2. **docs(setup)** — SETUP.md Node prerequisite 20+ → 22+, aligned with
   package.json engines (the single source of truth). nvm snippet updated.
3. **fix(tests)** — coverage thresholds bumped from 76/73/85/78 (stale) to
   88/80/89/91 (~1-2 pts below observed run). A real regression from 92% to
   85% lines now fails CI instead of passing silently.
4. **fix(mobile)** — Toast adds `max-w-[calc(100vw-2rem)]` to cap width on
   320px viewports. w-80 (320px) + right-4 gutter was overflowing.
5. **fix(dx)** — ESLint ignores `test-results/` + `playwright-report/`
   (prevents flaky lint when concurrent e2e writes there). Prerender script
   throws with actionable error if `pages.ts` is refactored in a way that
   breaks regex extraction (instead of silently producing empty meta).

## Decisions taken

- **React 19 `inert` over DOM-removal** for closed Accordion panels. The
  height animation requires DOM persistence; `inert` is exactly the right
  primitive for "rendered but unreachable".
- **Coverage thresholds 1-2 pts below observed**, not AT observed. Absorbs
  per-run v8 coverage jitter without hiding real drops.
- **Prerender stays regex-based** but fails loud on drift. Using `tsx` to
  import TS natively was considered — rejected to avoid a new devDep for
  a script that runs fine with simple regex on stable data shapes.
