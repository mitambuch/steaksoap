## Frictions

- **Regex foot-gun** in the Node-version extractor first pass — would have
  shipped broken had I not manually run doctor.js before committing. Lesson:
  always spot-check a regex that touches user-supplied string parsing.
- **Lint-staged reformatted multi-line arg in `git commit`** — prettier put
  the two-line test.skip on three lines. Harmless, accepted.

## Decisions taken

- **Dependabot fixes: all via overrides**, no direct dep bump for transitives.
  Keeps the reasoning localized in package.json. `pnpm why` confirms resolution.
- **SEO guard at build time, not runtime**: runtime warn already existed (did
  nothing in a CI pipeline). Build-time throw catches the issue where it matters.
- **Memory schema validation = error-level, not warn**: the memory system is
  load-bearing now. Silent drops = invisible failures. Better to fail loud.
