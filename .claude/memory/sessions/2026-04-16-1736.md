## Frictions

1. **Commit subject > 72 chars** on first try for the truth commit. Had to
   shorten. Pattern: write the commit message DRAFT in a comment block first,
   copy the subject line, eyeball-count. 50-char is the sweet spot.
2. **Regex lookahead failure** in rules-section extraction when there was
   no trailing `## ` heading. Caught by a new test (`flags rules present
   on disk but missing from the table`) failing. Fixed via substring
   indexing. Lesson: if a regex must work at end-of-file, don't use
   `(?=next boundary)` — use index math.

## Decisions taken

- **Table drift = detect-only, not auto-fix**. Adding a new command
  means picking a category in the table; the script can't safely guess
  placement. Auto-fix only where the transformation is unambiguous
  (heading counts). Tables and README phrases get a clear error message
  + exit 1 in check mode, forcing a human edit.
- **README phrase validation via regex, not JSON schema**. The phrase is
  human-facing prose; overspecifying it (every adjective, comma) would
  be brittle. A simple `(\d+)\s+slash commands.*(\d+)\s+contextual rules`
  catches the counts without caring about surrounding wording.
