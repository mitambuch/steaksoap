insisté sur 3 points qui sont devenus piliers de la rule :

- **Taxonomie stricte** : inline dans `page` / menu dédié avec picto pour

  pour le partagé. Zéro duplication.
- **Contenu depuis Claude Code** : on n'ouvre jamais le Studio à la main
  lors du dev initial. `/wire-content` rédige FR + DE + EN et pousse
  directement via `@sanity/client` (dataset staging par défaut).
- **Zéro champ vide** : `pnpm validate` fail si un singleton canonique
  a un `localeString` avec une locale manquante. Règle #13.
