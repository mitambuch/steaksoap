livraison client.

## Ce qui a shippé (v5.8.1, PR #42, 1 commit feat)

### Dynamic tagline — le problème steaksoap.app
- `src/data/pages.ts` passait de `tagline: 'v5.0 — production ready'`
  hardcodé (3 releases de retard sur main) à
  `` tagline: `v${__APP_VERSION__} — production ready` `` qui se
  remet à jour à chaque build. `__APP_VERSION__` est injecté par
  `vite.config.ts` via `define` depuis `package.json`.
- Impact : Netlify deploy auto tire `main`, rebuilds, tagline suit.

### Studio preview URLs
- `studio/sanity.config.ts` ajoute `document.productionUrl` :
  - `siteConfig` → `${PREVIEW_URL}/fr`
  - `page` → `${PREVIEW_URL}/fr/${slug}?draft=1` (`home` → slug vide)
- Bouton "View" en haut à droite du Studio ouvre le draft rendu.
- Configurable via `SANITY_STUDIO_PREVIEW_URL` / `SANITY_STUDIO_SITE_URL`
  ou fallback localhost.

### validate-protected-sync.js
- Script regex-parse les `PROTECTED = [...]` dans `base-patch.js` et
  `setup.js`, fail si divergence. Currently 7 paths in sync.
- Chaîné dans `pnpm validate`.

### pnpm handoff
- `scripts/handoff.js` génère `dist-handoff/handoff.html` printable.
- Lit `.claude/client.md` pour extraire le nom du client (regex
  ligne-par-ligne tolérante aux placeholders).
- Sortie : URLs site + Studio, first-login steps, top-3 actions
  client, astuces multilingue, bloc support.
- `--print` flag ouvre le fichier en fin d'exécution.

### Memory closure
- `.claude/memory/decisions/2026-04-18-content-stack-complete.md` —
  décision qui clôt le plan 5 phases avec tableau des releases, les 6
  invariants enforced, mapping des 13 leçons HDVA → guards.

  touche le fichier entre mes opérations. Fix : re-Read avant re-Edit.

  trim + placeholder detection.
