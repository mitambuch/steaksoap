# Session — Phase 3 app-side wiring + v5.7.0 shipped

Continuation de la triple-session 2026-04-18 (v5.5.0 → v5.6.0 → v5.7.0).
Phase 3 complète : l'app React est maintenant 100 % locale-aware,
SEO-multilingual-ready, et Sanity-ready via des hooks typés.

## Ce qui a shippé (v5.7.0, PR #40, 1 commit feat)

### LocaleProvider + useLocale
- `src/app/LocaleProvider.tsx` — context reads `:locale` URL param,
  syncs i18next + `document.documentElement.lang` + localStorage,
  expose `setLocale` qui navigue vers le même path sous la nouvelle
  locale. Fallback gracieux hors provider (tests / error boundaries).

### Router `/:locale/*`
- `src/app/LocaleRedirect.tsx` — un-prefixed paths Navigate vers locale-
  prefixed (detected).
- `src/app/routes/index.tsx` — canonical tree sous `/:locale`, 
  `LocaleLayout` valide le segment (redirect vers default locale si
  invalide, préservant path — résout `/unknown` → `/fr/unknown` → 404).
- `src/config/i18n.ts` helpers : `isLocale()`, `getInitialLocale()`,
  `localePath()`.

### SeoHead multilingue
- `<link rel="alternate" hreflang>` pour chaque SUPPORTED_LOCALES +
  `x-default`. Canonical pointe sur l'URL locale-prefixée. `og:locale`
  dynamique (fr_FR, de_DE, en_US).

### Sitemap multi-locale
- `vite.config.ts` Sitemap plugin `dynamicRoutes` = 3 paths × 3 locales
  = 9 URLs canoniques. Un-prefixed exclues.

### Hooks Sanity
- `src/hooks/useSanityDoc.ts` — générique `<T>(query, params?)` →
  `{ data, loading, error, retry }`. Disabled-shape synchrone quand
  `!hasSanity`.
- `src/hooks/useSiteConfig.ts` — wrapper typé pour le singleton.
- `src/types/siteConfig.ts` — `SanitySiteConfig` interface miroir du
  schéma Studio.

### Tests (+25 cases)
- `src/config/__tests__/i18n.test.ts` (13)
- `src/app/__tests__/LocaleProvider.test.tsx` (5)
- `src/app/__tests__/LocaleRedirect.test.tsx` (2)

## Frictions en session

- **React Router 7 `:locale` param greedy** — `<Route path="/:locale">`
  matche n'importe quelle string, y compris `/unknown`. Initial test
  `renders NotFound for unknown routes` fail (Home rendu au lieu de
  NotFound). Fix : `LocaleLayout` valide le segment et redirect vers
  `/:fallback/pathname` si invalide.
- **`react-hooks/set-state-in-effect`** — mon useEffect dans
  useSanityDoc appelait `setLoading(false)` directement. Fix : initial
  state déjà correct (`useState(hasSanity)`), early return dans l'effet.
- **`react-refresh/only-export-components`** — LocaleProvider.tsx export
  à la fois `LocaleProvider` (composant) et `useLocale` (hook). Fix :
  eslint-disable-next-line ciblé + commentaire WHY. Alternative
  (splitting) overkill pour le gain DX.
- **Import sort après mock vi** — mon 1er useSanityDoc.test.ts mettait
  imports après `vi.mock()` avec disable-comments `import/order` —
  règle inexistante dans la config. Fix : imports au top, `vi.mock`
  hoisté automatiquement par Vitest, aucun disable nécessaire.
- **Coverage branches en dessous du ratchet** — `useSanityDoc` à 16.66%
  parce que `hasSanity=false` en test env = happy-path fetch jamais
  exercé. Fix : 2e test file avec `vi.mock('@lib/sanity')` renvoyant
  `hasSanity=true` + `sanityClient.fetch = mock`.
